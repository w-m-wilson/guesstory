# Contributing

Thanks for your interest in contributing to Guesstory.

## Reporting bugs

Please open a GitHub issue. Include steps to reproduce, what you expected to happen, and what actually happened. Screenshots are helpful for UI bugs.

## Suggesting features

Open an issue describing the feature and why it would improve the game. Please do this before writing code — it avoids wasted effort if the direction isn't a fit.

## Pull requests

1. Open an issue first to discuss the change.
2. Fork the repo and create a branch from `main`.
3. Make your changes and verify the build passes (`npm run build`).
4. Open a pull request referencing the issue.

## Puzzle submissions

If you'd like to suggest a puzzle, open an issue with the category, the ranked list, and a source link.
